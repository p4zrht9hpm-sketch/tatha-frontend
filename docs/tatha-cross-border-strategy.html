3. Internationalization Roadmap

3.1 Architecture Decision: Migrate to Next.js

Current static HTML cannot scale to multi-language, multi-region deployment. Next.js with App Router provides:

Next.js App Router

Built-in i18n routing (/en/..., /ja/...), SSR for SEO, Edge Functions for region detection, middleware for locale redirect.

next-intl

Mature i18n library for Next.js. Supports message catalogs per locale, server/client components, ICU message format for plurals/gender.

Vercel Edge

Geo-based edge functions for region detection (IP → currency/pricing), already on Vercel so zero migration cost for hosting.

8. Platform Decision: Shopify vs SaaS-Native

During planning, the team evaluated Shopify as a potential platform for Tatha's international expansion. This section documents the analysis and final decision.

8.1 Why Shopify Was Considered

Multi-market presence

Shopify powers 4.8M+ stores globally across 175+ countries. Well-known brand reduces buyer hesitation.

Built-in payments

Shopify Payments supports 135+ currencies with integrated tax calculation. Single dashboard for all markets.

Quick setup

Templates and drag-and-drop builder can get a store live in days. Low technical barrier to entry.

8.2 Core Problem: Product vs Service Model Mismatch

Feature Fit Analysis — Shopify vs Tatha Requirements
RequirementTatha NeedsShopify ProvidesFit
Subscription management (recurring billing) Core — monthly/annual plans, usage tiers, proration Limited — Shopify Subscriptions API designed for physical product replenishment (e.g. "coffee of the month"), not SaaS tiers with feature gating No
User authentication + authorization Core — login, register, social OAuth (Google/GitHub/LINE), RBAC per plan Shopify accounts are customer accounts for checkout, not user accounts for SaaS access. No RBAC, no feature gating. No
API-driven AI interactions Core — real-time AI inference (resume parsing, job matching, MBTI), streaming responses No server-side logic. Shopify is a storefront, not an application platform. No WebSocket/SSE support for streaming AI. No
Usage metering + feature gating Core — "3 free AI analyses per day for Basic", "unlimited for Pro" Shopify tracks inventory and orders, not API call counts or usage quotas. No feature flagging system. No
Product catalog management Not needed — Tatha has no SKUs, no variants, no physical inventory Core strength — SKU management, variants, inventory tracking Irrelevant
Shopping cart + checkout Not needed — Users subscribe to a tier, not add items to cart Core strength — multi-item cart, discount codes, gift cards Irrelevant
Shipping / fulfillment Not needed — Pure digital delivery, no physical goods Core strength — carrier integration, tracking, multi-warehouse Irrelevant
Multi-currency + local payment Needed — 135+ currencies, regional payment methods Supported via Shopify Payments — but only within Shopify's checkout flow Partial
Verdict: Shopify solves zero of Tatha's core SaaS requirements (subscription management, auth, AI integration, feature gating). The 3 features Shopify excels at (product catalog, cart, shipping) are completely irrelevant to Tatha's business model. Forcing Tatha onto Shopify would require building custom apps for every core feature, essentially fighting the platform instead of leveraging it.

8.4 When Shopify Would Be Appropriate

Shopify is not wrong for all Tatha-related projects. Here are scenarios where it would fit:

ScenarioFitNotes
Physical merch store (T-shirts, planners, career books) Good fit If Tatha brand grows and wants to sell career-related physical products, Shopify is the right tool. But this is a Phase 3 side business, not the core.
One-time digital downloads (PDF resume templates) Possible Shopify can sell digital downloads. But Gumroad or LemonSqueezy are lighter-weight for this. Or just sell via Stripe Payment Links.
Brand landing page / marketing site Overkill Next.js + Vercel is more flexible, faster, and cheaper for a marketing site. No need for Shopify's commerce overhead.
Tatha core SaaS product (AI resume + job matching + MBTI) Wrong tool Not designed for subscription SaaS with AI features, auth, and usage metering.

8.5 Decision Matrix Score

Platform Fit Scoring (1-10, 10 = perfect fit)
Final decision: SaaS-native stack (Next.js + Stripe + Supabase) wins decisively with 8.9/10 vs Shopify's 3.4/10. Shopify is the right answer for product commerce, but Tatha is a subscription SaaS product. The two models require fundamentally different infrastructure.

Bookmark for later: If Tatha expands into physical merch or one-time digital downloads in Phase 3, revisit Shopify (or Printful API directly) for that specific use case — but never for the core SaaS product.

Video courses on job hunting, salary negotiation, LinkedIn optimization.

9.2 Physical Goods (Phase 3 - Optional)

If expanding to physical career-related products (books, planners, merch):

ModelHow It WorksBest ForMargin
